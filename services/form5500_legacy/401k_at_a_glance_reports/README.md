# 401k At-A-Glance Report Generation Pipeline

This repository contains a modular, production-grade pipeline for generating **“401k At-A-Glance” PDF reports** used for employer retirement plan benchmarking and prospect analysis.

The system pulls plan and company data from Supabase and Airtable, computes derived 401k metrics and benchmark ranks, formats the results, fills a PDF template, flattens the PDF, and attaches the final report back into Airtable.

---

## 📌 Overview of the Pipeline

The report generation flow consists of **five required steps**, executed in this exact order:

1. **build_401k_dataset.py**  
   Pulls raw plan, contribution, fee, and participation data from Supabase.  
   Enriches with company metadata from Airtable.  
   Produces the initial `401k_data.csv`.

2. **process_401k_dataset.py**  
   Cleans, normalizes, renames, and derives core report metrics.  
   Updates `401k_data.csv`.

3. **compute_benchmark_ranks.py**  
   Loads the enriched CSV.  
   Fetches benchmarking JSON from Supabase (`f_5500.retirement_benchmarking_summary`).  
   Computes generosity, contributions, and fees index ranks.  
   Saves ranks into `401k_data.csv`.

4. **format_401k_dataset.py**  
   Performs the **required final formatting pass**, including:  
   - Currency formatting  
   - Percentage formatting  
   - Whole-number strings  
   - N/A handling  
   Updates `401k_data.csv`.

5. **generate_and_attach_401k_reports.py**  
   Reads the fully prepared CSV and the PDF acroform template.  
   Fills all PDF fields, flattens the PDF, uploads it to a temporary host, and attaches the finished report to Airtable.

## 🗂️ File Dependencies

### **Local Files**

| File | Purpose |
|------|---------|
| `401k_at_a_glance_acroform.pdf` | PDF template used for generating final reports |
| `401k_data.csv` | Central dataset transformed across all steps |
| Generated PDFs | Final output attached back to Airtable |
| Benchmarking JSON (Supabase) | Consumed by `compute_benchmark_ranks.py` |

### **External Services**

- **Supabase Postgres**  
  Primary data source (401k/5500 tables) and home of benchmarking JSON.

- **Airtable**  
  Stores company metadata and receives the final PDF attachments.

- **tmpfiles.org (or similar)**  
  Temporary hosting used by the PDF upload process.

## ▶️ Running the Pipeline

Run each step **in order** from the project root:

```bash
python build_401k_dataset.py
python process_401k_dataset.py
python compute_benchmark_ranks.py
python format_401k_dataset.py
python generate_and_attach_401k_reports.py
```

After step 5, each Airtable record will have a fully populated, flattened “401k At-A-Glance” PDF.

## 📊 Benchmarking Data Note

This project relies on benchmarking JSON stored in:

```
f_5500.retirement_benchmarking_summary
```

This benchmarking data is generated by **external benchmarking tools** stored outside this repository.  
This project **only reads** the benchmarking JSON; it does not generate or upload it.
